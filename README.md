# pgwebl - acara 4

**pgwebl** is a web application built using Laravel framework (or a similar PHP framework). It is designed to provide a clean, modern, and responsive experience for users to interact with various features of the application.

## Features

- **Authentication**: Provides login, registration, password reset, and email verification functionality.
- **Admin Dashboard**: Includes features like user management, data tables, and management of polygons and points.
- **Real-time Updates**: Broadcasts real-time events to the users.
- **CRUD Operations**: Create, Read, Update, and Delete operations for managing data like points, polygons, and more.
- **Responsive Design**: Optimized for both desktop and mobile devices.

## Installation

1. **Clone the repository**:

    ```bash
    git clone https://github.com/fauziiladhim1/pgwebl.git
    cd pgwebl
    ```

2. **Install dependencies**:

    ```bash
    composer install
    npm install
    ```

3. **Set up environment variables**:

    Copy the `.env.example` file to `.env`:

    ```bash
    cp .env.example .env
    ```

    Then update the `.env` file with your database and other environment settings.

4. **Generate the application key**:

    ```bash
    php artisan key:generate
    ```

5. **Migrate the database**:

    ```bash
    php artisan migrate
    ```

6. **Run the application**:

    ```bash
    php artisan serve
    ```

    Visit `http://127.0.0.1:8000` in your browser to view the application.

## File Structure

- **app/**: Contains all the application logic including Controllers, Models, and Providers.
- **bootstrap/**: Contains application bootstrapping files.
- **config/**: Holds configuration files for the application.
- **database/**: Includes migrations, factories, and seeders for database tables.
- **public/**: The web server’s document root, contains assets like CSS, JS, images, and index.php.
- **resources/**: Contains views, raw assets, and language files.
- **routes/**: Contains web and API route definitions.
- **storage/**: Holds files generated by the application such as logs and cache files.
- **tests/**: Contains automated tests for the application.
- **vendor/**: Contains all the third-party packages installed via Composer.
